How to Read Rubik's Cube Algorithms:

There is some jargon in the world of Rubik's Cubes and some of them could be R, L, U, D, F, and B
To read an algorithm you first need to break it down. This is what each leter stands for: R = right L = left U = Upper D = Down F = Front B = back.
So if an algorithm as an upper cased U, you would turn the top face of the cube clock wise. If there is a U with and a apostrope you will turn the upper face counter clockwise. If there is a U and a 2 you will turn the face twice (it doesn't matter counter/clockise). This applies to every letter.
There is one last exception though. If the letter is lower cased you will turn the outer moster layers of that letter. So if you have an u, you will turn the two most upper layers together. If it has an apostrophe next to the u, yo will turn the two upper most layers counter clockwise.

OLL:

OLL stands for orientation of last layer. This means it will change the top face to all of that color.

Orientation of the Edges

First, you need the cross on the last layer
It it as dot you do the backwards L algorithm then the flat line algorithm
If it looks like a backwards L you put the L in the top left so one (yellow) edge is on the top and on the left. Then you do: F U R U' R' F'
If it looks like a flat line straight across you first put the line horizontally. This the algorithm you do to get the cross: F R U R' U' F'
If the yellow cross is on the top don't do anything

Orientation of the Corners

Now that we have the cross done we need to finsh up the OLL. There are 7 algorithms from the cross and here they are:
If there are two pairs of head lights face one pair of headlights facing you and the yellow side on top: R U2 R' U' R U R' U' R U' R'
If there is one pair of head lights and the rest is solved put the headlights facing you and the yellow side on top: R2 D R' U2 R D' R' U2 R'
If there is one pair of head lights and the rest of the corners is not solved but not a headlight put the head light on the left: R U2 R2 U' R2 U' R2 U2 R
If there are no headlights but two corners next to each other are solved, put the none solved corners on the left: r U R' U' r' F R F'
If there are two solved corners diagonal to each other put one in the bottom left and one in the top right, but put the corner that is not solved that will be facing you with the solved corners in the bottom left and right: F' r U R' U' r' F R
Now there's sune, put the solved corner in the bottom left and make sure the corner in the bottom right the yellow is facing you: R U R' U R U2 R'
Now there's anti-sune, put the solved corner in the back right then do: R U2 R' U' R U' R'

PLL

PLL stands for permutation of last layer, so pretty much we are finally putting everything into place. First, we need to permutate the corner peices then the edges.

Permutation of Corners

Okay if there is one pair of head lights that are on the last layer, put those in the back of the cube, then do: R' F R' B2 R F' R' B2 R2 U'
If all the corners are all permutated you can skip this step. And if there are no head lights do the algorithm above then there will be a pair of headights.

Permutaion of Edges

This is the last step. If there is one edge permutated with its corner pieces out that in the back then do: F2 U R' L F2 R L' U F2
If this does not solve it do that algorithm one more time. And if at first there are no edges permutated with its corner do the algorithm once then find the permutated edge.
